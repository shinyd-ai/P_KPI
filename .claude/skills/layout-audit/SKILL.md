# Layout Audit Skill

## 목적

이 Skill은 responsive-analyzer가 모바일 레이아웃 문제를 빠뜨리지 않고 찾아내도록 체크 항목을 제공한다.

## 절차

1. **구조 파악**: `app/layout.tsx` 를 읽어 전체 레이아웃 골격 확인
2. **사이드바 확인**: `components/layout/Sidebar.tsx` 에 모바일 분기(`md:hidden` 등) 있는지 확인
3. **고정 너비 탐색**: Grep으로 `w-\d`, `min-w-`, `max-w-` 패턴 검색
4. **flex/grid 확인**: 반응형 prefix 없이 쓰인 `flex`, `grid-cols-` 탐색
5. **버튼/터치 확인**: `py-1`, `p-1` 처럼 터치 영역이 너무 작은 버튼 탐색
6. **오버플로 확인**: `overflow-hidden` 없이 가로로 넘칠 수 있는 구조 탐색

## 검색 패턴 (Grep 기준)

```
고정 너비:    w-\d+(?!\/), min-w-\d
반응형 미적용: ^(?!.*md:).*flex-row
터치 영역:    py-1\b, p-1\b
```

## 산출물 형식

문제를 심각도 3단계로 분류한다.

- **높음**: 모바일에서 레이아웃이 바로 무너지는 것
- **중간**: 좁은 화면에서 어색하거나 스크롤이 생기는 것
- **낮음**: 사용은 가능하지만 개선하면 더 좋은 것

## 품질 기준

- 파일명과 줄 번호를 반드시 함께 표기한다
- 추측이 아니라 읽은 코드를 근거로 작성한다
- 분석 보고서는 설계자가 바로 설계에 사용할 수 있어야 한다
