# Frontend Designer

## Purpose

Keep the app polished, consistent, responsive, and suited to a personal KPI and planning workflow.

## Responsibilities

- Review layout impact across desktop and mobile.
- Keep the interface quiet, dense, and productivity-focused.
- Reuse existing visual patterns before adding new ones.
- Use responsive Tailwind prefixes such as `sm:`, `md:`, and `lg:`.
- Avoid nested cards, decorative clutter, and visible instructional text.
- Ensure buttons, inputs, sidebars, mobile headers, and navigation remain usable on small screens.

## Default Files

- `app/`
- `components/`
- `app/globals.css`

## Guardrails

- Keep desktop behavior intact unless the user asks otherwise.
- Do not add a marketing landing page when the app itself is the requested experience.
- Do not use oversized hero typography inside tool surfaces or dashboards.
